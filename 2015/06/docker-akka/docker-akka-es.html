Docker es un proyecto de código abierto que automiza el despliegue de aplicaciones al interior de contenedores de software basados en Linux. Este post nos muestra como desplegar una aplicación scala-akka usando docker.

Configurando nuestro entorno docker

Instalando el cliente docker 

    wget https://get.docker.com/builds/Linux/x86_64/docker-latest
    sudo ln -s /home/manuel/src/dockermachine/docker-latest /usr/bin/docker

Instalando Docker Machine

Docker Machine permite crear docker host en nuestras máquinas locales, en la nube o en nuestros centros de datos: 

    mkdir ~/src/docckermachine
    cd ~/src/docckermachine
    wget  https://github.com/docker/machine/releases/download/v0.3.0/docker-machine_linux-amd64
    sudo ln -s /home/manuel/src/dockermachine/docker-machine_linux-amd64 /usr/bin/docker-machine

Ahora crearemos un host docker en virtualbox el cual correrá una versión ligera de linux (boot2docker) con el proceso docker instalado con el siguiente comando: 

    $docker-machine create --driver virtualbox myhost

    To see how to connect Docker to this machine, run: docker-machine env myhost

Nota: Necesitamos tener instalado virtualBox para crear el docker host, su instalación es sencilla y va más alla del alcance de este post .

Podemos verificar si la creación del host fue exitosa: 

    $ docker-machine ls
    NAME    ACTIVE   DRIVER       STATE     URL                         SWARM
    myhost   *        virtualbox   Running   tcp://192.168.99.100:2376

Ahora podemos apuntar nustro cliente docker al host docker que hemos creado: 

    eval "$(docker-machine env myhost)"

Podemos acceder al host docker con el siguiente comando: 

    docker-machine ssh myhost

To stop and start docker machine: 

  docker-machine stop myhost
  docker-machine start myhost

Podemos probar nuestra instalación ejecutando el contenedor nginx: 

     docker run -d -p 8000:80 nginx
     docker ps

Instalando una imagen

Podemos listas las imagenes instaladas en nuestro host docker ejecutando el siguiente comando: 

    $ docker-machine ls
    NAME     ACTIVE   DRIVER       STATE     URL   SWARM
    myhost            virtualbox   Stopped

Podemos descargar una imagen debian desde Docker hub e instalarla en nustro host docker con el siguiente comando: 

    docker pull debian:latest

Servidor tcp basado en Scala y Akka toolkit

El código a continuación implementa un servidor tcp echo usando reactive streams: 

    package io.ntdata

    import akka.actor.ActorSystem
    import akka.pattern.ask
    import akka.stream.ActorFlowMaterializer
    import akka.stream.scaladsl.{ Flow, Sink, Source, Tcp }
    import akka.util.ByteString
    import scala.concurrent.duration._
    import scala.util.{ Failure, Success }

    object TcpEcho {

      /**
       * Use without parameters to start both client and
       * server.
       *
       * Use parameters `server 0.0.0.0 6001` to start server listening on port 6001.
       *
       * Use parameters `client 127.0.0.1 6001` to start client connecting to
       * server on 127.0.0.1:6001.
       *
       */
      def main(args: Array[String]): Unit = {
        if (args.isEmpty) {
          val system = ActorSystem("ClientAndServer")
          val (address, port) = ("127.0.0.1", 6000)
          server(system, address, port)
          client(system, address, port)
        } else {
          val (address, port) =
            if (args.length == 3) (args(1), args(2).toInt)
            else ("127.0.0.1", 6000)
          if (args(0) == "server") {
            val system = ActorSystem("Server")
            server(system, address, port)
          } else if (args(0) == "client") {
            val system = ActorSystem("Client")
            client(system, address, port)
          }
        }
      }

      def server(system: ActorSystem, address: String, port: Int): Unit = {
        implicit val sys = system
        import system.dispatcher
        implicit val materializer = ActorFlowMaterializer()

        val handler = Sink.foreach[Tcp.IncomingConnection] { conn =>
          println("Client connected from: " + conn.remoteAddress)
          conn handleWith Flow[ByteString]
        }

        val connections = Tcp().bind(address, port)
        val binding = connections.to(handler).run()

        binding.onComplete {
          case Success(b) =>
            println("Server started, listening on: " + b.localAddress)
          case Failure(e) =>
            println(s"Server could not bind to $address:$port: ${e.getMessage}")
            system.shutdown()
        }

      }

      def client(system: ActorSystem, address: String, port: Int): Unit = {
        implicit val sys = system
        import system.dispatcher
        implicit val materializer = ActorFlowMaterializer()

        val testInput = ('a' to 'z').map(ByteString(_))

        val result = Source(testInput).via(Tcp().outgoingConnection(address, port)).
          runFold(ByteString.empty) { (acc, in) â‡’ acc ++ in }

        result.onComplete {
          case Success(result) =>
            println(s"Result: " + result.utf8String)
            println("Shutting down client")
            system.shutdown()
          case Failure(e) =>
            println("Failure: " + e.getMessage)
            system.shutdown()
        }
      }
    }

Podemos usar el plugin SBT Assembly para generar el fatjar que desplegaremos en el contenedor docker: 

    stb assembly

Iniciar el servidor es sencillo: 

    java -jar target/scala-2.11/ReactiveTCP-assembly-0.0.1.jar server 0.0.0.0 6001

Construyendo una imagen a partir de un archivo Dockerfile

Para crear una imagen debemos crear un archivo llamado Dockerfile, este archivo contendrá las instrucciones requeridas para construir nuestra imagen:

mkdir akka-image touch akka-image/Dockerfile cp tcp-server/target/scala-2.11/ReactiveTCP-assembly-0.0.1.jar akka-image

Nuestro archivo Dockerfile: 

    # Debian based image that is able to run an akka app
    FROM debian:latest
    MAINTAINER Manuel Ignacio Franco Galeano 

    #install oracle jdk
    RUN echo oracle-java8-installer shared/accepted-oracle-license-v1-1 select true |  /usr/bin/debconf-set-selections 

    RUN echo "deb http://ppa.launchpad.net/webupd8team/java/ubuntu trusty main" | tee /etc/apt/sources.list.d/webupd8team-java.list

    RUN echo "deb-src http://ppa.launchpad.net/webupd8team/java/ubuntu trusty main" | tee -a /etc/apt/sources.list.d/webupd8team-java.list
    RUN   apt-key adv --keyserver hkp://keyserver.ubuntu.com:80 --recv-keys EEA14886
    RUN   apt-get update -y
    RUN   apt-get install -y oracle-java8-installer 
    RUN   apt-get install -y oracle-java8-set-default

    USER daemon

    # Adding fatjar file
    ADD ReactiveTCP-assembly-0.0.1.jar /opt/

    # Exppose port where server will be running
    EXPOSE 6001

    CMD [ "java", "-jar", "/opt/ReactiveTCP-assembly-0.0.1.jar", "server", "0.0.0.0",  "6001" ]

Debemos ejecutar docker build -t simpletcp:v1 akka-image/ para generar la imagen.

Luego de la creación de la imagen podemos crear el contenedor ejecutando docker run -d -p 6000:6001 simpletcp:v1
