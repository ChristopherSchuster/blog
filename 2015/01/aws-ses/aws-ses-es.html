Amazon SES es un servicio que permite a ususarios de AWS users enviar correo electrónico. Este post es una introducción a este servicio.

Verificando tu email

AWS requiere la verficación de cada email que se desee usar como remitente de correos, el proceso es simple, instrucciones detalladas pueden ser encontradas en el siguiente enlace: Verificando direcciones de en Amazon SES

>Asignar permisos de envio a un grupo

Es buena idea tener diferentes grupos para diferentes tareas, a continuación una política de seguridad que otorga acceso a SES para un grupo: 

    {
        "Version": "2012-10-17",
        "Statement":[{
           "Effect":"Allow",
           "Action":["ses:SendEmail", "ses:SendRawEmail"],
           "Resource":"*"
           }
        ]
     }

Para asignar esta política a un grupo podemos ejecutar el siguiente comando: 

    $aws iam put-group-policy --group-name webapp --policy-document file://ses-policy.json --policy-name ses-webapp

AWS-cli y SES

Podemos usar la consola para acceder a los servicios SES, para ver una lista de comandos disponibles podemos usar el comando aws ses help . Lo primero que podemos hacer es listar las identidades asociadas a una cuenta con el comando aws ses list-identities : 

    $aws ses list-identities

    {
        "Identities": [
            "email@mydomail.com"
        ]
    }

Enviando mensajes 

aws ses send-email --from email@mydomail.com --to user@box.com --subject 'test message' --text 'hello world'  --html '
hello world
'
